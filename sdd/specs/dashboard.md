# Specification: Dashboard

## الهدف

الداشبورد ليس عرض أرقام فقط. هو واجهة تفسير:

- ماذا يتوقع النموذج؟
- لماذا؟
- ماذا حدث فعلاً؟
- كيف تغير النموذج بعد ذلك؟

## التبويبات

1. المباريات والتوقعات.
2. المجموعات والمسارات.
3. مواجهات محتملة.
4. دقة النموذج.

## بطاقة المباراة

يجب أن تعرض:

- المنتخبين بالعربية.
- ألوان المنتخبين.
- مرحلة المباراة.
- احتمالات الفوز/التعادل/الخسارة.
- العوامل المؤثرة.
- تحذيرات نقص البيانات.

## نافذة التفاصيل

عند الضغط على المباراة:

- توقع ما قبل المباراة.
- النتيجة الفعلية عند توفرها.
- هل أصاب النموذج؟
- Brier Score.
- Log Loss.
- رسوم إحصاءات المباراة.
- العوامل التي رجحت التوقع.

## Acceptance Criteria

- كل مباراة لها بطاقة.
- كل بطاقة قابلة للضغط.
- تفاصيل المباراة لا تعتمد على API مباشر؛ تقرأ ملفات JSON/CSV المعالجة.
- إذا لم توجد إحصاءات، يظهر تحذير واضح.
